## 三、数据库与安全

**当前方案**：付费 cpolar 隧道 + 24 小时 NAS 上的 MySQL。该方案**可用于正式版**——云函数是服务端出站连接，不受小程序合法域名/ICP 备案限制；付费隧道地址固定、NAS 常开，早期/中小规模可行。以下为加固与安全项（保留为上线前必做）。

- [ ] ⚠️ **连库云函数超时设为 20s**（新建函数默认可能 3s，冷启动 + 隧道延迟易超时，之前 -504003 即此因）
- [ ] ⚠️ **轮换 MySQL 密码**（旧密码曾进 git 历史）：改 `.env` → `npm run sync-db` → 重新部署云函数
- [ ] **cpolar 隧道加访问认证 / IP 白名单**，NAS 防火墙只放行必要端口，避免本地库被公网随意连接
- [ ] **NAS 上 MySQL + cpolar 客户端设开机自启 + 断线自动重连**；建议接 UPS 防瞬时断电
- [ ] （可选，看规模）用户量 / 稳定性要求上来后，再迁移到腾讯云 MySQL（同机房内网低延迟、有 SLA）

> 说明：家用网络毕竟无机房级 SLA（停电/宽带故障/运营商维护会导致全站下线，且无冗余），是自托管方案固有的单点风险，早期可接受。
