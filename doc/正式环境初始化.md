# 正式环境（prod）初始化 Runbook

> 一次性操作，务必在**发版窗口**内进行（新版小程序已过审、待点「发布」）。
> 策略（2026-07-19 定）：**只搬「表结构 + 静态字典数据」，业务数据单独初始化，云存储暂不迁移。**

## 背景与要点

- **dev 与 prod 是同一 MySQL 实例上的两个库**：`xingshu_dev`（开发/体验）、`xingshu_prod`（正式），同一 cpolar 隧道、同一账号。故一条连接即可跨库操作。
- **CloudBase 两环境**（2026-07-13 对调后）：dev = `cloud1-xingshu-prd-d1cev0fcca864`；prod/release = `cloud1-d9gbozhfp4a6c50c0`。
- **为何不再逐条跑 4 个迁移脚本**：`migrate-diary-to-story`、`migrate-activity-price`、`migrate-admin-roles`、`migrate-post-video` 是用来把**旧 prod 逐步演进**到现结构的。既然直接按 **dev 现行结构**重建 prod，prod 一步到位拿到最终结构，**这 4 个脚本对全新 prod 初始化不再需要**（仅当有人保留旧 prod 增量迁移时才用）。
- **云存储暂不迁移**：图片/视频/头像的 fileID 仍指向 dev 环境；prod 业务数据初始化后如需显示历史媒体，另行迁移（改版 `migrate-storage.js`，方向 dev→prod）。

## 初始化会做什么（`scripts/init-prod.js`）

1. 安全护栏：`xingshu_prod.users` 非空则拒绝（防误清生产库，确认覆盖加 `--force`）。
2. DROP 并按 `xingshu_dev` **现行结构重建** prod 全部表（17 张）。
3. 仅复制**字典表数据**：`tags`（20）、`activity_types`（8）。
4. 其余表（users/stories/activities/orders/interactions/comments/...）**只建结构、留空**。

## 执行步骤

```bash
# 0. 确认在发版窗口；确认 prod.users 为空（脚本亦会护栏）
node scripts/init-prod.js --dry            # 先演示，确认源表与字典表无误

# 1. 备份 prod（习惯动作，即便为空）
XINGSHU_ENV_FILE=.env.prod node scripts/backup-db.js

# 2. 初始化 prod 库（结构 + 字典）
node scripts/init-prod.js                  # prod.users 为空时直接执行；已有数据需 --force

# 3. 部署 prod 云函数（脚本内部会先据 .env.prod 生成各函数 db.js/secret.js）
bash scripts/deploy-functions.sh --prod

# 4. ⚠ 关键：把本地 db.js/secret.js 恢复回 dev，避免本地测试误连 prod
npm run sync-db

# 5. admin 后台构建并发布到 prod（release 环境）
cd admin && npm run build && cd ..
npx -y -p @cloudbase/cli tcb hosting deploy admin/dist -e cloud1-d9gbozhfp4a6c50c0

# 6. 小程序：微信开发者工具切 release 环境，上传 → 后台点「发布」
```

## 收尾

- **业务数据单独初始化**：会员、故事、活动等按业务侧方案导入/开通（不在本 Runbook 范围）。
- **首个运营账号**：admin 后台用全局密码（`.env.prod` 的 `ADMIN_PASSWORD`，经 sync-db:prod 生成到 secret.js）登录超管，再在「账号管理」建运营账号。
- **云存储**：后续需要历史媒体时再迁移。

## 风险与注意

- `init-prod.js` 破坏性：会清空重建 prod 全部表。首次 prod.users 为空，安全；重跑需 `--force` 且会清掉业务数据。
- 部署 prod 云函数后**务必 `npm run sync-db`** 恢复 dev 配置（deploy-functions.sh 不会自动回滚本地 db.js）。
- `generateMiniCode` 依赖 openapi 权限，CLI 部署若扫码报 -604101，改用微信开发者工具对该函数「上传并部署」。
