## v1.6.0 — 2026-07-23

四页签大改版：新增醒书问答，活动页升级为首页。

**底部导航改为四页签（所有用户一致）**
- 醒书活动（首页）/ 醒书故事 / 醒书问答 / 醒书会员，四个页签所有身份统一显示。
- 「我的故事」「我的问答」「我的收藏」不再占页签，入口收进**醒书会员**页；非会员只保留「我的收藏」一行。

**新增「醒书问答」模块**
- 会员可发问（帖子式，只写正文），可选**匿名**；其他会员回答，回答也可匿名。
- 匿名发布**不显示头像和昵称**——头像为默认「醒」字、昵称显示「醒书同学」；**同一串问答里用头像底色区分不同的人**（同一人恒定同色）。
- 授权用户可点赞、收藏问答；后台可将优质问答**纳入精选**开放给非会员浏览（可读全部回答但不能回复）。
- 问题详情页问题前加「问：」标记；回答输入改为**多行文本框**，适合写长回答。

**活动页升级为小程序首页**
- 顶部新增**可手动拨动的 Banner 轮播**，由后台管理（上传图、可选跳转图文详情页）。
- 活动列表每行信息更完整：**起止时间**、**形式**（线下显示活动地点 / 线上向已报名者显示会议号）、**共创组织者**；活动类型标签移到卡片右上角。
- 活动详情页增加**共创组织者**信息，并支持**收藏活动**（收藏中心新增「活动」段）。
- 筛选优化：类型与时间（按月）改为**就地下拉面板**，Banner 与筛选栏固定、仅列表滚动。
- 隐藏了「活动分享」子栏目（功能保留，后续可恢复）；现场分享的发布权收窄为主理人/工作人员。

**醒书故事**
- 会员新增**精选筛选**（搜索栏的星标按钮）：只看运营精选的故事。
- 全站不再显示点赞数量，只显示自己是否已赞。
- 精选故事的对外版本不含评论区；评论收窄为会员专享。

**管理后台**
- 新增**问答管理**（含精选问答的纳入/修订/上下架）与**活动 Banner 管理**。
- Banner 详情页改为**所见即所得富文本编辑器**（画布按手机真实宽度呈现，支持插入图片）。
- 「善选」全面改称「精选」。

**数据安全**
- 数据库启用**每日自动加密备份**至云存储。

> 本版为破坏性大改版，prod 上线需执行 `migrate-v2` 迁移并全量部署云函数（新增 qa、改动 activity/admin/getStoryList/getStoryDetail/createComment）+ 重新构建发布管理后台。详见 `doc/devlog.md`。

## v1.5.7 — 2026-07-20

新增品牌启动页。

**每日首次打开的品牌页**
- 用户**每天第一次打开**小程序时，先看到一屏「醒書知行社」品牌页——「修身為本」印章、社区简介「一个探索把经典走进生活的社区」和「**我要进入**」按钮，点一下进入；同一天内再次打开不再出现，次日恢复。
- 点按钮后有「**钤印 → 画面推开**」的开启动效，与印章主视觉呼应。
- 从**扫码或好友转发**直接打开某篇故事、某场活动时同样会出现，且**盖在那篇内容上**——点进入后直接看到对方分享的内容，不会跳走，不影响分享转化。
- 从后台切回小程序（热启动）不会重复弹出。
- 视觉规格取自原设计原型，与全站纸质暖色调、印章体系一致。

> 该页顺带遮住了冷启动时故事列表尚未返回的空白期，等待被包装成有内容的品牌画面。纯前端改动，无数据库或云函数变更，prod 上线无需额外迁移。

## v1.5.2 — 2026-07-19

后台多运营角色权限体系上线 + 活动报名数据移动端查看（小程序新页面）。

**后台多运营角色（管理后台）**
- 新增**运营账号体系**：运营人员用**手机号+密码**登录后台（超管在「账号管理」建账号、设初始密码、可停用/重置密码）；原管理员密码登录保留为超管入口。
- **四种角色**：超级管理员（全部功能）/ 内容运营（故事/善选/互动）/ 活动运营（仅自己主理的活动）/ 会员运营（用户/订单，不含退费）。**一个账号可同时授多个角色**（权限取并集）。
- 菜单与页面按角色显示；越权操作服务端拒绝；所有操作审计记录真实操作者。
- 侧边栏显示**当前登录者姓名与角色**。

**活动主理人与工作人员**
- 每个活动可指定**主理人**（超管设置）；活动运营角色只能管理自己主理的活动。
- 主理人可在后台为活动添加**工作人员**（白名单）。

**小程序 · 报名数据页（新页面）**
- 活动主理人与工作人员在活动详情页可打开「**报名数据**」：报名/到场/已收费统计 + 名单全量（称呼、联系方式、报名时间、到场/收费状态），报名人有推荐人的一并显示**推荐人昵称与 ID**。
- 页面可**转发给工作人员**作为入口，打开时按身份鉴权（不在名单只见提示页）；数据只读，到场/收费勾选仍在后台。

**故事分享海报优化**
- 海报**高清导出**（配图按原图分辨率生成，不再模糊）。
- 故事标题**单行自适应**（自动缩字号，尽量不换行）。

> 运营/部署侧：本版含新表 admin_accounts / activity_staff 与 activities.owner_user_id，prod 上线需先执行 `migrate-admin-roles` 迁移，并部署 admin、activity 云函数与管理后台；小程序新页面 activity-stats 需随本版过审发布。上线后由超管创建运营账号、为存量活动补设主理人。

## v1.5.0 — 2026-07-18

「日记」全面升级为「故事」，上线后台善选机制与作者数据视角（重大版本）。

**日记升级为故事 · 写作两态**
- 全线文案与内容实体由「日记」改为「故事」。
- 会员写故事简化为**暂存 / 发布**两个按钮：暂存仅自己可见，发布后面向会员。原来的三选一权限（公开/会员/私密）与「会员墙截断」下线。

**善选机制 · 广场按身份分流**
- 公众能看到什么，改由运营在后台「善选」决定：按热度榜人工挑选故事纳入善选，生成可单独修订的副本（不影响作者原文）。
- **会员**：广场浏览全部已发布故事的原文；**未登录 / 已登录非会员**：只看善选故事（副本全文）。

**善选故事免登录可读 · 互动才需登录**
- 善选故事**未登录也能进详情读全文**（运营认可的对外内容，不再被登录墙挡住传播）。
- 点赞 / 收藏 / 评论 / 回复 / 保存海报，在点击那一刻才拉起微信登录，登录后自动接着完成。

**底部页签按身份增减**
- 未登录 3 个（广场 / 活动 / 会员中心）；登录后加「我的收藏」；会员再加「我的故事」。登录、退出、开通会员后即时增减。

**分享收敛 · 全文海报**
- 只有**善选故事**提供分享海报，海报含副本全文与故事配图，底部为醒书咨询品牌栏（含可扫小程序码）。非善选故事只走右上角原生转发。

**我的故事页 · 作者数据视角**
- 「我的故事」卡片新增阅读数，并可点开阅读 / 点赞 / 收藏 / 评论四类**人员清单**（谁、什么时候、评论内容）。
- 广场、详情、收藏页的卡片不再显示收藏数字（收藏操作保留）。

**故事徽章**
- 眼睛 = 已发布，金星 = 已入选善选，折角 = 暂存稿。

> 运营/部署侧：本版为破坏性升级（表 / 云函数改名 + 善选副本表 + 阅读记录表），旧版小程序在新版发布后不可用（已加强制更新）。上线需在过审后、点发布前的窗口内完成数据库迁移与全量云函数部署，详见 devlog 上线清单。
